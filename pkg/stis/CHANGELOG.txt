### 28-Apr-2026 - CALSTIS Version 3.5.0:
    - Implement improved wavecal calibration for CCD data taken at the E1 or E2 aperture locations.  
      The part of the long slit wavecal spectrum that matches the location of the source is used to 
      calculate the SHIFTA1 value.
    - Common timestamp
    - Minor improvements:
        - The rootname could be NULL so now it's only reported when there is something valid
        - Safely pads - characters to the end of the message string
    - Fixes conditions leading to segfaults due to buffer overflows, mostly from trailer file
      functions (trlerror, trlwarn) attempting to report messages larger than the line buffer. 
      This also adjusts the size of several buffers to accommodate the expected input, otherwise 
      they would likely be truncated at run-time.

### 24-Feb-2014 - CALSTIS Version 3.4:
    - In the section for echelle scattered light correction, function GetPSF reads 
      the PSF image from a reference table (_tel.fits).  If the aperture that was 
      used for the exposure is smaller than the PSF image, GetPSF then truncates 
      the PSF image to include only the portion that would have gotten through the 
      aperture.  The PSF image is oversampled quite a bit, and it needs to be binned 
      down to match the STIS image. The number of STIS image pixels for the PSF 
      (truncated by the aperture) is found from the aperture size and plate scale, 
      but this computation did not take into consideration the possibility that 
      the aperture may actually be larger than the PSF image.  This has been fixed 
      by modifying the two lines for computing ms and ml.

### 25-Jan-2013 - CALSTIS Version 3.0:
    - The TDCTAB is now required for NUV-MAMA data.
    - DarkMsg in do2d.c prints info about the TDC table.
    - darkHistory in history1.c writes a HISTORY record for the TDC table.
    - In dodark.c, function medianDark was added.  This is called for NUV-MAMA data, 
      and the value (median of the binned dark reference image) is passed 
      as a new argument to GetTdcCorr.
    - For imaging data, calstis has been adding three dummy keywords to the DQ 
      extension header, in order to work around a bug in the IRAF FITS kernel. 
      The code to do this has been removed from the hstcal version of calstis.
    - Fixed issue where two input images were opened but never closed during
      cosmic ray rejection
    - Removed hard-coded limit of 120 on number of input image sets. Arrays whose
      size was set by this number now get set to the actual number of input image sets.
    - Add function PrFullVersion to print the full version string, and call this 
      function in each of the calstis modules (except cs6) for command-line option -r.  
      cs6 was not modified because it already has -r to specify the maximum search size.
    - Update the date string, for the change to allocate file name template strings 
      based on the length of the command-line argument.
    - Increase the buffer size for file names to 1025.
    - When calstis1 constructs the photmode string (aka obsmode, the value for keyword PHOTMODE) 
      for imaging data, "MIRROR" is now included in the string.
    - For the photcorr step with imaging data, explicitly apply a correction for the variation 
      in detector sensitivity with temperature.
    - The FLUX and ERROR columns will both be set to 0 if pedigree is dummy. 
      A new NET_ERROR column will be added to the x1d table (regardless of pedigree), 
      which contains the error estimate of the NET column, both of which are in counts per second.
    - Calstis will now accept either the original STIS format for a TDS table or a COS-like TDS table.
    - The photcorr correction checks for photometry parameters of -9999, which indicates that the 
      date of observation is outside the range of the imphttab. 
    - The fluxcorr for calstis6 sets the flux to zero if the relevant row in the phottab is dummy.
    - Added a check that there was at least one input file specified.  Added a check for 
      "strange" (e.g., INF) values of the phot parameters.
    - For each calstis module, the function that reads the command-line arguments now 
      checks for "--version", and if present, the function prints the calstis version number 
      (without the date) and exits with status 0.
    - The following changes are for making use of the IMPHTTAB and functions
      InitPhotPar, GetPhotTab, and FreePhotPar:
        - For imaging data, check for IMPHTTAB instead of PHOTTAB.
        - In PhotMsg, keyword phottab was changed to imphttab, and info is no longer printed 
          about apertab or tdstab.
        - Functions InitPhotPar, GetPhotTab, and FreePhotPar are called, instead of 
          explicitly getting the throughput info and calling c_phopar.
        - Function checkPhot was modified to check for keyword IMPHTTAB instead of PHOTTAB, 
          and keywords APERTAB and TDSTAB are ignored. 
        - Function photHistory was modified to not check filtcorr or tdscorr.
        - The Modified Julian Date was added to the photmode string (MJD#<date>).
        - cs2_command returns a status code, but it wasn't being checked.  A test on the value 
          returned by cs2_command was added; if it's not zero, exit is called with that value of status.
    - The following changes make this version consistent with the IRAF-dependent version of CALSTIS 2.36:
        - CALSTIS should use reference temp and slope from CCD dark file
        - Add another term to the dispersion relation in CALSTIS.
        - Rewrite calstis function for inverting the dispersion relation.
        - Improve the a4corr correction for echelle wavecal processing in calstis.
        - Calstis failure when an imset is missing (exptime was 0).
    - CALSTIS was using hard-coded parameters for the reference temperature and slope in order to 
      correct the temperature dependence of the CCD dark rate.  Analysis of SMOV data shows that 
      these values have changed. Rather than including time-dependent parameters in calstis, these are 
      now included as header keywords in the dark reference files, and calstis has been modified to 
      read and use these keyword values.
    - A ninth term has been added to the dispersion relation.  
      This is proportional to m**2, where m is the spectral order.
    - Improved the a4corr correction for echelle wavecal processing by deriving shifta2 from inputs
      instead of taking the moffset2 value as an estimate. The a4corr correction is a modification 
      to the echelle dispersion coefficients to account for a skew that results from a displacement 
      of the image in the cross-dispersion direction.
    - Fixed issue where call to c_tbtclo failed while checking whether an image set exists or not.
